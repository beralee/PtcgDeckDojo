## 对战设置页卡组查看/编辑入口设计

### 目标

在对战设置页中，为 `玩家1卡组` 和 `玩家2/AI卡组` 各自提供独立的 `查看`、`编辑` 按钮：

- `查看`：复用卡组管理页中的卡组详情查看效果
- `编辑`：进入现有卡组编辑器
- 编辑完成后：直接返回对战设置页，而不是跳回别的页面

### 交互设计

- `玩家1卡组` 右侧显示 `查看`、`编辑`
- `玩家2/AI卡组` 右侧显示 `查看`、`编辑`
- 当对应选择器没有有效卡组时，按钮禁用
- 点击 `查看` 时，打开与卡组管理页一致的详情弹窗
- 点击 `编辑` 时，跳转到当前所选卡组的编辑器
- 编辑器返回后，恢复到对战设置页

### 返回流

编辑卡组时，需要保存一个最小的返回上下文：

- 来源场景为 `BattleSetup`
- 当前 `玩家1/玩家2` 所选卡组 id
- 当前模式、先后手、AI 选项、背景、音乐等设置

返回对战设置页后：

- 重新刷新卡组列表
- 尝试按返回前记录的 deck id 恢复选择
- 其余对战设置保持不变

### 实现边界

- 不复制 `DeckManager` 的完整查看/编辑实现
- 卡组详情查看逻辑抽成可复用 helper，供 `DeckManager` 和 `BattleSetup` 共用
- `编辑后返回对战设置页` 通过 `GameManager` 的最小返回上下文实现
- 不改动对战开始逻辑
- 不改动卡组编辑器内部业务逻辑，只改返回目的地

### 测试

至少覆盖：

- 对战设置页两边各自存在 `查看 / 编辑` 按钮
- 未选中有效卡组时，对应按钮禁用
- 点击 `查看` 能打开卡组详情
- 点击 `编辑` 会进入编辑器
- 编辑器返回后回到对战设置页，并刷新/恢复卡组选择
