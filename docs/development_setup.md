# 开发环境与运行说明

## 1. 环境要求

- Godot `4.6.x`
- Windows 环境下已验证
- 需要可写的本地用户数据目录用于保存 `user://` 缓存

项目当前将卡牌 JSON、卡图和卡组数据缓存到本地用户目录，而不是仓库目录。

## 2. 打开项目

1. 启动 Godot
2. 导入当前仓库目录
3. 打开 `project.godot`
4. 运行主场景 `res://scenes/main_menu/MainMenu.tscn`

## 3. 主要流程

### 卡组导入

1. 进入“卡组管理”
2. 输入 `tcg.mik.moe` 卡组链接或 deck ID
3. 导入器会顺序拉取卡组详情、缺失卡牌详情并同步卡图

### 对战启动

1. 导入至少两个卡组
2. 进入“开始对战”
3. 选择模式和双方卡组
4. 进入战斗主场景

## 4. 本地缓存说明

项目使用 `user://` 保存本地数据，核心目录包括：

- `user://cards/`
- `user://cards/images/`
- `user://decks/`

这意味着：

- 仓库本身不需要提交缓存后的卡牌 JSON 或卡图
- 清理本地用户目录会导致重新下载或重新导入
- 发布开源仓库时可以保持“代码入库、缓存不入库”

## 5. 测试入口

### 运行完整测试场景

项目通过 `res://tests/TestRunner.tscn` 汇总主要测试套件。

示例命令：

```powershell
& 'D:\ai\godot\Godot_v4.6.1-stable_win64_console.exe' --headless --path 'D:\ai\code\ptcgtrain' 'res://tests/TestRunner.tscn'
```

如果你本机的 Godot 路径不同，只需要替换可执行文件路径。

### 重点测试类别

- 数据模型测试
- 规则与状态机测试
- 效果系统与注册测试
- 卡牌语义矩阵测试
- UI 回归测试
- 编码审计测试

## 6. 编码规范

仓库对编码问题采取零容忍策略：

- 所有文本文件统一为 UTF-8
- 批量改写或脚本写盘后必须重新检查中文显示
- 文档修改也应视为需要验证的改动

相关约束以 [../DEVELOPMENT_SPEC.md](../DEVELOPMENT_SPEC.md) 为准。

## 7. 开发建议

- 修改卡牌效果前，先确认是否已有可复用效果类
- 需要玩家选择的效果，优先接入统一交互步骤框架
- 不要把本地测试缓存、临时日志或 AI 助手配置提交到公开仓库

## 8. 当前公开仓库建议

如果你准备把仓库作为公开项目维护，建议优先保持以下边界：

- 公开代码、测试和设计文档
- 不公开用户缓存和卡图
- 不把内部助手配置和临时工作文件纳入版本控制
- 在 README 中明确声明项目为非官方练牌/开发项目
